# Steps to setup Drill4J

This page describes general steps required to setup Drill4J for your project.<br/>

**Use it as a checklist and open specific pages for detailed instructions depending on your project needs.**<br/>

:::note
Before diving into setup, read the [Overview](/docs/overview/why-drill4j) section to get a better grasp on reasons to use Drill4J
:::

1. **Step 1** Deploy Drill4J services

2. **Step 2** Integrate application(s) agent

   - Java or any JVM language
   - Frontend JavaScript/TypeScript

3. **Step 3** Integrate with tests

   - Manual (in Chrome or Chromium-based browsers)
   - E2E (with Selenium or Cypress)
   - API (in Java, JavaScript or Postman)

Read further for detailed instructions.

## Step 1 - Deploying Drill4J services

Refer to [Deploy Drill4J services](/docs/installation/deploy-drill4j-services) section

## Step 2 - Integrating application agent

      { label: 'Node.js', value: 'node-js' },

  >
    <TabItem value="single-java">
      <ul>
        <li>
          You have a <b>single backend application/service</b> written in Java (or other JVM language, e.g. Scala, Groovy, etc)
        </li>
        <li>
          You are running tests just for this service or you are not interested in metrics for other parts of your system (e.g. frontend)
        </li>
      </ul>
      In this case you need to setup <a href="/docs/installation/setup-java-agent">Java Agent</a>
    </TabItem>
    <TabItem value="single-javascript">
      <ul>
        <li>
          You have a <b>web application </b> written in JavaScript / TypeScript or any other flavor of JS
        </li>
        <li>You are running tests just for this app or you are not interested in metrics for other parts of your system (e.g. backend)</li>
      </ul>
      In this case you need to setup <a href="/docs/installation/setup-javascript-agent">JavaScript Agent</a>
    </TabItem>
    <TabItem value="java-js">
      <ul>
        <li>
          You have a <b>Java backend</b> and <b>JavaScript frontend</b>
        </li>
        <li>
          You are running some sort of e2e/UI tests (e.g. with Selenium or Cypress) affecting both services and would like to get metrics
          for the entire system
        </li>
      </ul>
      In this case you need to:
      <ol>
        <li>
          setup <a href="/docs/installation/setup-java-agent">Java Agent</a>
        </li>
        <li>
          setup <a href="/docs/installation/setup-javascript-agent">JavaScript Agent</a>
        </li>
        <li>
          configure them to be in a <a href="/docs/installation/setup-service-group">Service group</a>
        </li>
      </ol>
    </TabItem>
    <TabItem value="microservices">
      You are running <b>a bunch of microservices / services in Java</b> and <i>maybe</i> also <b>a JavaScript frontend</b>
      <br />
      <br />
      In this case you need to:
      <ol>
        <li>
          setup <a href="/docs/installation/setup-java-agent">Java Agent</a> for each of backend services (though it's optional and you may
          start with just one)
        </li>
        <li>
          configure them to be in a <a href="/docs/installation/setup-service-group">Service group</a>
        </li>
        <li>
          if you also have a frontend app in Javascript, you should setup{' '}
          <a href="/docs/installation/setup-javascript-agent">JavaScript Agent</a> and add it to the Service Group as well
        </li>
      </ol>
    </TabItem>
    <TabItem value="node-js">Currently, we do not support Node.js</TabItem>
  </Tabs>
</div>
