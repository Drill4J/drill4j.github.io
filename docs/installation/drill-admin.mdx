## Using Jar
### Drill Admin Backend

1. Go to artifactory: <a href='https://drill4j.jfrog.io/artifactory/drill/com/epam/drill/admin-core/' target="_black">
    https://drill4j.jfrog.io/artifactory/drill/com/epam/drill/admin-core/
<ExternalLinkIcon/>
</a>


2. Find the necessary version and download it. You need a file with the name: **admin-core-'verison'.jar**.

<AdditionalInformation className="space-y-2">

E.g. admin-core-0.7.0.jar
</AdditionalInformation>

3. Run it:
```java
 java -jar admin-core-0.7.0.jar
```

4. Drill Admin Backend is on <a href='http://localhost:8090/apidocs/index.html?url=./openapi.json' target="_black">
    localhost:8090
<ExternalLinkIcon/>
</a>

## HTTPS Setup
If your application uses **https** for access you need to deploy additional **ssl-proxy** service
to Drill admin setup:

```yaml
ssl-proxy-admin:
    image: drill4j/ssl-proxy:0.1.0
    ports:
      - 8443:8443
    environment:
      DOMAIN: drill4j.example.com
      SSL_PORT: 8443
      TARGET_HOST: admin-ui
      TARGET_PORT: 8080
```

For example, full docker-compose file looks like:

```java
version: '3'

services:
ssl-proxy-admin:
    image: drill4j/ssl-proxy:0.1.0
    ports:
      - 8443:8443
    environment:
      DOMAIN: drill4j.example.com
      SSL_PORT: 8443
      TARGET_HOST: admin-ui
      TARGET_PORT: 8080

  drill-admin:
    image: drill4j/admin:0.7.0
    environment:
      - TEST2CODE_PLUGIN_VERSION=0.7.0
      - LOG_LEVEL=INFO
    ports:
      - 8090:8090

  admin-ui:
    image: drill4j/admin-ui:0.7.1
    ports:
      - 8091:8080
```
Drill Admin is available on **https://drillAdminHost:8443**

<AdditionalInformation className="space-y-2">

Accept self signed certificate

</AdditionalInformation>
