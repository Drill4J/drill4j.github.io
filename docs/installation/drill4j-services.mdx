# Deploy Drill4J services

The easiest way to run Drill4J services is using [Docker](https://www.docker.com/).

1. Create `docker-compose.yml` file with the following contents

   ```yaml
   version: '3'

   services:
     # admin panel UI
     admin-ui:
       image: drill4j/admin-ui-root-config:${FRONT_VERSION}
       ports:
         - 8091:8080
       environment:
         WAIT_HOSTS: drill-admin:8090
         UI_PLUGINS_URLS: 'test2code#https://cdn.jsdelivr.net/npm/@drill4j/test2code-ui@${TEST2CODE_UI_VERSION}/dist/Drill4J-test-to-code.js'
       networks:
         - drill4j-network
       depends_on:
         drill-admin:
           condition: service_healthy

     # main backend service
     drill-admin:
       image: drill4j/admin:${BACK_VERSION}
       environment:
         - TEST2CODE_PLUGIN_VERSION=${TEST2CODE_PLUGIN_VERSION}
         - JAVA_TOOL_OPTIONS="-Dplugin.feature.drealtime=true"
         - LOG_LEVEL=debug # pass "trace" value for extended log when troubleshooting
         - DRILL_DB_PORT=5432
         - DRILL_AGENTS_SOCKET_TIMEOUT=6000 # increase if you experience agent disconnects during long operations
         - DRILL_DB_HOST=${POSTGRES_HOST}
         - DRILL_DB_NAME=${POSTGRES_DB}
         - DRILL_DB_USER_NAME=${POSTGRES_USER}
         - DRILL_DB_PASSWORD=${POSTGRES_PASSWORD}
         - DRILL_DB_MAX_POOL_SIZE=${DRILL_DB_MAX_POOL_SIZE}
         - LOG_MESSAGE_MAX_LENGTH=4000 # comment out to get a complete log when troubleshooting
       healthcheck:
         test: ['CMD', 'curl', 'http://localhost:8090']
         interval: 1s
         timeout: 3s
         retries: 30
       ports:
         - 8090:8090
         - 6006:5006
       networks:
         - drill4j-network
       volumes:
         - drill-data:/work
       depends_on:
         postgres:
           condition: service_healthy

     # main db
     postgres:
       image: postgres:14.1
       ports:
         - '5432:5432'
       environment:
         POSTGRES_PASSWORD: ${POSTGRES_PASSWORD}
         POSTGRES_USER: ${POSTGRES_USER}
         POSTGRES_DB: ${POSTGRES_DB}
       volumes:
         - drill-data-pg:/var/lib/postgresql/data
       healthcheck:
         test: ['CMD-SHELL', 'pg_isready -U postgres']
         interval: 10s
         timeout: 5s
         retries: 5
       networks:
         - drill4j-network

     # a "proxy" that allows to interact with Chrome's CDP API via HTTP instead of Websocket
     # required for Selenium-based tests
     # comment out if you don't need that
     devtools-proxy:
       image: drill4j/devtools-proxy:${DEVTOOLS_PROXY_VERSION}
       ports:
         - '8093:8080'
       environment:
         DEBUG: 'drill:*'
         DEBUG_COLORS: 'true'
         FORCE_COLOR: '3'
         DEBUG_LOG_LEVEL: '2'
       extra_hosts:
         - 'host.docker.internal:host-gateway'

     # component required to process coverage from Web JavaScript applications (both from manual and automated tests)
     # comment out if you dont need that
     js-agent:
       image: drill4j/js-agent:${JS_AGENT_VERSION}
       ports:
         - '8092:8092'
       environment:
         APP_PORT: '8092'
         WAIT_HOSTS: drill-admin:8090
         DRILL_ADMIN_PROTOCOL: 'ws'
         DRILL_ADMIN_HOST: 'drill-admin:8090'
         # variables bellow are optional (comment out to silence the output log)
         DEBUG: 'drill:*'
         DEBUG_COLORS: 'true'
         FORCE_COLOR: '3'
         DEBUG_LOG_LEVEL: '4'
       volumes:
         - drill-data-js-agent:/usr/src/app/js-agent-data
       networks:
         - drill4j-network
       depends_on:
         drill-admin:
           condition: service_healthy

   volumes:
     drill-data:
     drill-data-pg:
     drill-data-js-agent:

   networks:
     drill4j-network:
   ```

2. Create the following `.env` file next to it

   ```shell
   # drill-admin / postgres
   BACK_VERSION=0.8.0-79
   TEST2CODE_PLUGIN_VERSION=0.8.0-78
   POSTGRES_HOST=postgres
   POSTGRES_DB=postgres
   POSTGRES_USER=postgres
   POSTGRES_PASSWORD=mysecretpassword
   DRILL_DB_MAX_POOL_SIZE=10

   # admin-ui
   FRONT_VERSION=0.1.0-147
   TEST2CODE_UI_VERSION=0.1.0-135

   # devtools-proxy
   DEVTOOLS_PROXY_VERSION=0.1.0

   # js-agent
   JS_AGENT_VERSION=0.3.0-beta.4
   ```

3. Open command line and run `docker-compose up -d`

> **Q**: Is there a way to run Drill4J services without docker?
>
> **A**: Yes, but it is cumbersome and would require you to
>
> - install Java 8, PostgreSQL, Node.js 16 (depending on exact setup).
> - manually deploy and configure all services listed in docker-compose file above.
>
> So it's best to use Docker images provided by Drill4J out-of-the-box
