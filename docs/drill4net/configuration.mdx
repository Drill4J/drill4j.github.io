This page describes configuration of *Test Agent* and *Drill4Net Scanner* Applications.

## Test Agent Configuration

*Test Agent* can be configured using drill4netsettings.yml file which can be found in *Test Agent* application directoty.
Table below describes each setting.

<Table>

| Setting                           | Description                                                                                                                                                                                                                                                    |
|-----------------------------------|----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Id**                            | Integration tests client. Currently only HTTP client is supported, so this setting should be **HttpClient**.                                                                                                                                                   |
| **TestAssemblySourcePath**        | Path to the assembly with integration tests, including filename. Path to build with integration tests, including filename. This value can also be specified as an argument to the Test Agent application.                                                      |
| **AdminAgentConfig:Id**           | Arbitrary agent name.                                                                                                                                                                                                                                          |
| **AdminAgentConfig:BuildVersion** | Version of the build. Should be new for every new build.                                                                                                                                                                                                       |
| **CommunicatorUrl**               | Communication URL of Drill4J Admin back-end.                                                                                                                                                                                                                   |
| **Serilog:***                     | Logger settings. Detailed information about Serilog configuration is available [here](https://github.com/serilog/serilog-settings-configuration). Note: not all settings are available as we may not have required Serilog package for specific functionality. |

</Table>

Example of the drill4netsettings.yml:
```
Settings:
  Client: HttpClient
  TestAssemblySourcePath: d:\Builds\AppUnderTest.Tests\netcoreapp3.1\AppUnderTest.Tests.dll

Serilog:
  Using:
    - Serilog.Sinks.Console
  MinimumLevel: Debug
  WriteTo:
    - Name: Console

AdminAgentConfig:
  Id: 'appId-1'
  BuildVersion: '0.0.1'
CommunicatorUrl: 'http://localhost:8090'
```

## Drill4Net Scanner Configuration

*Drill4Net Scanner* can be configured using appsettings.yml file which can be found in *Test Agent* application directoty.
Table below describes each setting.

<Table>

| Setting                           | Description                                                                                                                |
|-----------------------------------|----------------------------------------------------------------------------------------------------------------------------|
| **adminAgentConfig:id**           | Arbitrary agent name.                                                                                                      |
| **adminAgentConfig:buildVersion** | Version of the build. Should be new for every new build.                                                                    |
| **communicatorUrl**               | Communication URL of Drill4J Admin back-end.                                                                                         |
| **schedulerDelayInterval**        | Interval in seconds indicating how often code coverage data is sent to the Drill4J Admin. The default value is 10 seconds. |

</Table>

Note that values of `adminAgentConfig:id`, `adminAgentConfig:buildVersion` and `communicatorUrl` should be same as in appsettings.yml of 
[Drill4Net.Injector.App (Test Agent)](#test-agent-configuration). Example of the *Drill4Net Scanner* appsettings.yml:
```
adminAgentConfig:
  id: 'appId-1'
  buildVersion: '0.0.1'
communicatorUrl: 'http://localhost:8090'
schedulerDelayInterval: 10
```
