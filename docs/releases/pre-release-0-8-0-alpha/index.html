Known Issues#

  • Reconnecting the Admin deletes the active session
  • There is no ability to collect code coverage into active session after Agent restarting (need to finish it and start a new session)
  • If you change Project Packages or the Target host, the Finished Scopes are lost
  • Tests duration for manual tests is displayed 00:00:00 always
  • Sometimes not all agents are displayed on the "Service Group: Test2Code" page
  • Sometimes active sessions are not correctly displayed in the "Sessions Management" pane
  • Opening multiple tabs of Drill4J Admin Panel may cause UI update issues. Please, consider using only one tab
